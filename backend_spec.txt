# **2\. Flyway Migrations (exact SQL)**

## **V1\_\_create_users.sql**

CREATE EXTENSION IF NOT EXISTS "pgcrypto";

CREATE TABLE users (

id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

username VARCHAR(24) NOT NULL UNIQUE,

color CHAR(7) NOT NULL,

created_at TIMESTAMPTZ NOT NULL DEFAULT now()

);

## **V2\_\_create_tiles.sql**

CREATE TABLE tiles (

id INTEGER PRIMARY KEY,

row SMALLINT NOT NULL,

col SMALLINT NOT NULL,

owner_id UUID REFERENCES users(id) ON DELETE SET NULL,

captured_at TIMESTAMPTZ,

UNIQUE(row, col)

);

CREATE TABLE capture_log (

id BIGSERIAL PRIMARY KEY,

tile_id INTEGER NOT NULL REFERENCES tiles(id),

user_id UUID NOT NULL REFERENCES users(id),

captured_at TIMESTAMPTZ NOT NULL DEFAULT now()

);

CREATE INDEX idx_tiles_owner ON tiles(owner_id);

CREATE INDEX idx_caplog_user ON capture_log(user_id);

CREATE INDEX idx_caplog_tile ON capture_log(tile_id);

## **V3\_\_seed_tiles.sql**

\-- Seeds 50×50 = 2500 tiles. Idempotent.

INSERT INTO tiles (id, row, col)

SELECT

r \* 50 + c AS id,

r AS row,

c AS col

FROM generate_series(0, 49) AS r

CROSS JOIN generate_series(0, 49) AS c

ON CONFLICT (id) DO NOTHING;

_If GRID_SIZE is changed from 50, update the SQL accordingly and add a new migration V4_\_.sql - never edit existing migration files.\_

# **4\. Key Service Implementations**

## **TileService.java - captureTile method**

@Service @RequiredArgsConstructor @Slf4j

public class TileService {

private final TileRepository tileRepo;

private final UserRepository userRepo;

private final RedisTemplate&lt;String,String&gt; redis;

private final SimpMessagingTemplate messaging;

@Value("\${app.cooldown-ms}") private long cooldownMs;

@Transactional

public TileDto captureTile(int tileId, UUID userId) {

// 1. Check cooldown

String cooldownKey = "cooldown:" + userId;

if (Boolean.TRUE.equals(redis.hasKey(cooldownKey))) {

Long ttl = redis.getExpire(cooldownKey, TimeUnit.MILLISECONDS);

throw new CooldownException(ttl != null ? ttl : cooldownMs);

}

// 2. Pessimistic lock

Tile tile = tileRepo.findByIdWithLock(tileId)

.orElseThrow(()-> new NotFoundException("Tile not found"));

// 3. Update

User user = userRepo.getReferenceById(userId);

tile.setOwner(user);

tile.setCapturedAt(Instant.now());

tileRepo.save(tile);

// 4. Set cooldown in Redis

redis.opsForValue().set(cooldownKey,"1",cooldownMs,TimeUnit.MILLISECONDS);

// 5. Publish event (broadcast happens via Redis listener)

TileDto dto = TileDto.from(tile);

redis.convertAndSend("tile:updated", toJson(dto));

return dto;

}

}

## **TileBroadcaster.java - Redis → STOMP bridge**

@Component @RequiredArgsConstructor @Slf4j

public class TileBroadcaster implements MessageListener {

private final SimpMessagingTemplate messaging;

private final LeaderboardService leaderboard;

@Override

public void onMessage(Message message, byte\[\] pattern) {

try {

TileDto tile = fromJson(message.getBody(), TileDto.class);

// Broadcast tile update

messaging.convertAndSend("/topic/tiles",

Map.of("type","TILE_UPDATED","payload",Map.of("tile",tile)));

// Broadcast leaderboard

messaging.convertAndSend("/topic/leaderboard",

Map.of("type","LEADERBOARD_UPDATE","payload",

Map.of("leaderboard",leaderboard.getTop10())));

} catch (Exception e) { log.error("Broadcast error",e); }

}

}

## **TileRepository.java - Pessimistic Lock Query**

@Repository

public interface TileRepository extends JpaRepository&lt;Tile, Integer&gt; {

@Lock(LockModeType.PESSIMISTIC_WRITE)

@Query("SELECT t FROM Tile t WHERE t.id = :id")

Optional&lt;Tile&gt; findByIdWithLock(@Param("id") int id);

@Query("SELECT t, u FROM Tile t LEFT JOIN t.owner u ORDER BY t.id")

List&lt;Tile&gt; findAllWithOwner();

}

# **5\. Global Exception Handler**

@RestControllerAdvice

public class GlobalExceptionHandler {

@ExceptionHandler(ConstraintViolationException.class)

ResponseEntity&lt;?&gt; handleValidation(ConstraintViolationException e) {

return ResponseEntity.badRequest()

.body(Map.of("error","INVALID_INPUT","message",e.getMessage()));

}

@ExceptionHandler(CooldownException.class)

ResponseEntity&lt;?&gt; handleCooldown(CooldownException e) {

return ResponseEntity.status(429)

.body(Map.of("error","COOLDOWN","remainingMs",e.getRemainingMs()));

}

@ExceptionHandler(UsernameConflictException.class)

ResponseEntity&lt;?&gt; handleConflict(UsernameConflictException e) {

return ResponseEntity.status(409)

.body(Map.of("error","USERNAME_TAKEN","message",e.getMessage()));

}

@ExceptionHandler(Exception.class)

ResponseEntity&lt;?&gt; handleGeneric(Exception e) {

log.error("Unhandled",e);

return ResponseEntity.internalServerError()

.body(Map.of("error","INTERNAL_ERROR","message","Something went wrong"));

}

}

# **8\. Deployment Guide**

**Follow these steps in order. Skipping steps causes hard-to-debug failures.**

## **8.1 Option A - VPS (Recommended, cheapest)**

Works on any Ubuntu 22.04 VPS (DigitalOcean Droplet, Hetzner CX21, Vultr, etc.). Minimum specs: 2 vCPU, 2 GB RAM.

### **Step 1 - Provision server**

- Create Ubuntu 22.04 VPS, minimum 2 GB RAM
- SSH in as root, create non-root user: adduser gridwar && usermod -aG sudo gridwar
- Set up SSH key auth, disable password login
- Point your domain A record to the VPS IP

### **Step 2 - Install Docker**

\# Run as root or with sudo

apt-get update && apt-get install -y ca-certificates curl

install -m 0755 -d /etc/apt/keyrings

curl -fsSL <https://download.docker.com/linux/ubuntu/gpg> \\

\-o /etc/apt/keyrings/docker.asc

echo "deb \[arch=\$(dpkg --print-architecture) \\

signed-by=/etc/apt/keyrings/docker.asc\] \\

<https://download.docker.com/linux/ubuntu> \\

\$(. /etc/os-release && echo \$VERSION_CODENAME) stable" \\

\> /etc/apt/sources.list.d/docker.list

apt-get update

apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

usermod -aG docker gridwar # so non-root can run docker

### **Step 3 - Deploy app**

\# As gridwar user

git clone <https://github.com/YOUR_ORG/gridwar> /opt/gridwar

cd /opt/gridwar

cp .env.example .env

nano .env # fill in real passwords + domain

docker compose -f docker-compose.yml -f docker-compose.prod.yml \\

up -d --build

\# Check all 4 containers are healthy:

docker compose ps

### **Step 4 - HTTPS with Caddy (recommended)**

apt-get install -y debian-keyring debian-archive-keyring apt-transport-https

curl -1sLf <https://dl.cloudsmith.io/public/caddy/stable/gpg.key> \\

| gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg

curl -1sLf <https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt> \\

| tee /etc/apt/sources.list.d/caddy-stable.list

apt-get update && apt-get install caddy

Create /etc/caddy/Caddyfile:

yourdomain.com {

reverse_proxy localhost:80

}

systemctl enable caddy && systemctl start caddy

_Caddy auto-provisions Let's Encrypt TLS. No certbot needed._

### **Step 5 - Verify**

\# Hit health endpoint

curl <https://yourdomain.com/actuator/health>

\# Should return: {"status":"UP"}

\# Open <https://yourdomain.com> in browser - GridWar should load

## **8.2 Option B - Railway (fastest, zero-config)**

- Push code to GitHub
- Go to railway.app → New Project → Deploy from GitHub
- Railway auto-detects Docker Compose - add all env vars in Railway dashboard
- Add PostgreSQL plugin and Redis plugin from Railway marketplace
- Update .env values to use Railway-provided connection strings
- Deploy - Railway gives you a public URL automatically with HTTPS

## **8.3 Option C - Render**

- Create a new "Web Service" from your GitHub repo
- Set build command: docker build -f frontend/Dockerfile -t frontend . (for frontend)
- Render has managed PostgreSQL and Redis - add them from dashboard
- Set all environment variables in Render dashboard
- Deploy - Render provides HTTPS URL

# **9\. Basic Monitoring**

## **Health Checks**

All services expose health endpoints used by Docker Compose depend-on conditions:

| **Service** | **Health endpoint**                    |
| ----------- | -------------------------------------- |
| Backend     | GET /actuator/health → {"status":"UP"} |
| PostgreSQL  | pg_isready -U gridwar                  |
| Redis       | redis-cli ping → PONG                  |
| Frontend    | nginx serves 200 on /                  |

## **Viewing Logs**

docker compose logs -f backend # Spring Boot logs

docker compose logs -f postgres # DB logs

docker compose logs -f frontend # nginx access logs

## **Database Backup**

\# Run daily via cron on VPS

docker exec gridwar-postgres-1 \\

pg_dump -U gridwar gridwar | gzip \\

\> /opt/backups/gridwar\_\$(date +%Y%m%d).sql.gz

# **10\. Backend Test Checklist**

All must be green before moving to Phase 4 (frontend integration).

- POST /api/users - 200 with UserDto, session cookie set
- POST /api/users - duplicate username → 409
- POST /api/users - invalid chars → 400
- GET /api/grid - returns exactly 2500 tiles
- PATCH /api/tiles/0 - no session → 401
- PATCH /api/tiles/0 - valid session → 200 with updated TileDto
- PATCH /api/tiles/0 - second call within cooldown → 429 with remainingMs
- PATCH /api/tiles/0 - after cooldown expires → 200
- WebSocket STOMP connect → receive TILE_UPDATED on /topic/tiles after another client captures
- docker compose up --build → all 4 services healthy within 60 seconds
